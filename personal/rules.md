# 个人规则（使用者增补层）

> 自进化的落点之二：词的问题进 [lexicon.md](./lexicon.md)，规则的问题进本文件。
> 执行 skill 主流程前必读本目录两个文件；本文件优先级高于 references/ 的通用规则。

## 维护规则（写给执行 skill 的 agent）

1. **显式反馈立即写**：用户明说一条规则（某类写法不好 / 某种表达要保持）时，立即在"条目"区追加，按模板格式
2. **隐式反馈先提炼再确认**：用户批评或亲手改写产出后，从中提炼表达偏好候选（词层进 lexicon.md，句式 / 风格层进本文件），在当轮收尾时列出候选（≤3 条，附改前 → 改后证据）问用户是否存入；确认哪条写哪条，不确认不写
3. 每条必须带日期和用户原话（或改前改后对照）——不许凭印象概括立规则，也不许凭"行业惯例"预填
4. 同类反馈追加到已有条目（更新日期、补原话），不开新条
5. 自进化只允许写 personal/ 下两个文件，不改 references/ 核心规则；一条规则经 2-3 篇产出反复验证后，可提议用户升级进核心分册，用户确认才动
6. 本文件条目与核心规则冲突时，以本文件为准

## 条目模板

```markdown
### 规则名（一句话，可执行）
- 日期：YYYY-MM-DD
- 用户原话："……"
- 规则：……（可带正例/反例）
- 状态：生效 | 已升级进核心（指向文件）
```

# 标题学（v1，三源交叉提炼：大厂汇报原稿 / 技术高管讨论型 PPT / 周枫文章）

标题规则按载体分三套，共用一条底线：**读者只读标题，应该得到什么？**——文章给全部论点，PPT 给叙事线，文档给骨架加承诺。

## 文章 / 对外材料：标题即论点

- 主标题用判断句或等式（"Harness即产品"），不用主题词（"关于 Harness 的思考"是弱标题）
- 小节标题全部写成判断句或行动句（"要做高智能产品""把上下文工程当成主任务"），连读 = 完整论点列表
- 检验：把全部标题抄出来给没读过正文的人看，他能复述你的主张吗？

## PPT：按类型分两套

- **讨论型**：页标题用短锚点串议程（The "Why" → Current Situation → Next Stage），锚点是路标不是结论，结论在讨论中产生
- **汇报型**：每页标题 = 该页结论句（"成本下降 40%"不是"成本分析"），标题连读 = 汇报主线

## 文档（规划 / 总结 / OKR）：双档节奏

大厂原稿实测规律——标题长度两档分布，中间几乎没有过渡：

- **骨架层（章节标题）**：4-12 字名词短语，只圈范围不下结论——"一、背景""二、研发效能""2-1：指标列表"。骨架标题不带数字
- **承诺层（O/KR/关键结果标题）**：40-90 字完整判断句，标题即结论即指标——"O1：【运营提效】……线上化率从 58%→72%，整体提升运营效率 13%"（数字已虚构，保留形态）。凡是表态的标题一律嵌数字（百分比 / 比值 / 区间）
- **KR 标签前缀法**：`KR序号：【承诺/挑战】【场景词】+ 动作 + 指标`——是可读性最高的承诺层格式；同一文档内标签与标点格式必须统一（反例：同份文件出现 `KR1:` `KR1：` 两种写法）

## 反模式（三源共同验证）

1. **框架型标题占结论位**："概述：阶段总结及计划"——首页标题位是全文最贵的位置，放框架标签等于浪费；结论应该上提进标题
2. **主题词标题**："XX 分析""XX 情况""XX 的思考"——读标题不知结论
3. **附录式弱标题蔓延到正文**："排兵布阵："这类只有标签没有信息的标题，只允许出现在附录
4. **长度停在中间档**：20-30 字的标题往往是"想下结论又没下完"——要么压到 12 字内做骨架，要么写满一句带指标的承诺

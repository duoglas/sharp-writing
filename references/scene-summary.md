# 总结 / 规划分册（项目总结 / 年度规划 / 复盘）

## 结构：What + How + Next 三段

- **What（结果）**：总分结构——开头总判断 1-3 句（每个维度各一句），然后按业务名词展开详细数据。三七开定性放段首（"成绩究竟是三分还是七分，必须有个根本的估计"），细节跟后
- **How（关键动作）**：3 件最有价值的动作 + 1-2 件需复盘的。每件"做了什么 + 价值/影响"，拒绝抽象判断（"从口号转案例驱动"不是动作）
- **Next（下一步）**：每件含里程碑 + 负责人 + 验收口径，缺一不可

## 写前四步（抓主要矛盾）

1. 列全本期所有事项，强制挑出对目标起决定作用的一件做主线——不做这步的下场是"如堕烟海，找不到中心"
2. 每件事内部先定性再展开（三分还是七分）
3. 定性必须有数量支撑：占比多少、及格线在哪、处在哪边
4. 主线详写，其余每条一句话点到状态（弹钢琴：凡有问题的地方都要点一下，但不展开）

## 论证结构（重要段落）

"提出一个问题 → 加以分析 → 综合起来指明性质 → 给出解决办法"。编号列表使用前先问：这几条之间有内部联系吗？有统领它们的判断吗？没有就不是分析，是中药铺式罗列。

## 复盘骨架（项目/季度复盘）

每条主线固定三段：

- **成果**：数字三件套（基线→实际→目标）；试点/首期结果只领"初步证明"，把"还不能证明什么"显式写出来（"初步证明联营模式可以盘活尾部网点；但试点仅两个月，尚不能证明可规模化复制"）
- **反思**：用"如果重来一遍，会做哪些改变？"逼出真话。承认投入打水漂的要给止损决策（"该侧投入较多但收效甚微，后续非核心能力直接内嵌，并接受部分体验下降"），不写"继续加强/持续关注"式空反思
- **沉淀**：这期留下了什么可复用的能力或结论，没有就写没有

每条主线末尾给一档总定性（红/黄/绿或三七开），放段首或段尾固定位置。

## 规划留痕（规划/方案类）

- **被否方案连同理由写出来**：方案对比两案并列、优缺点各列、标注推荐；没选的那个写清为什么不选（"不优化框架而重写的三条原因"）——下个读者不用重走一遍论证
- **没想清楚的地方显式标注**："待定 / 待讨论 / 待分析"直接写在正文里，不假装完整。标清边界比处处有答案更可信
- **目标分承诺/挑战双档**，各带数字——一个是底线一个是上限，读者知道该按哪个问责

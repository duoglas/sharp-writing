# 总结 / 规划分册（项目总结 / 年度规划 / 复盘）

## 结构：What + How + Next 三段

- **What（结果）**：总分结构——开头总判断 1-3 句（每个维度各一句），然后按业务名词展开详细数据。三七开定性放段首（"成绩究竟是三分还是七分，必须有个根本的估计"），细节跟后
- **How（关键动作）**：3 件最有价值的动作 + 1-2 件需复盘的。每件"做了什么 + 价值/影响"，拒绝抽象判断（"从口号转案例驱动"不是动作）
- **Next（下一步）**：每件含里程碑 + 负责人 + 验收口径，缺一不可

## 写前四步（抓主要矛盾）

1. 列全本期所有事项，强制挑出对目标起决定作用的一件做主线——不做这步的下场是"如堕烟海，找不到中心"
2. 每件事内部先定性再展开（三分还是七分）
3. 定性必须有数量支撑：占比多少、及格线在哪、处在哪边
4. 主线详写，其余每条一句话点到状态（弹钢琴：凡有问题的地方都要点一下，但不展开）

## 论证结构（重要段落）

"提出一个问题 → 加以分析 → 综合起来指明性质 → 给出解决办法"。编号列表使用前先问：这几条之间有内部联系吗？有统领它们的判断吗？没有就不是分析，是中药铺式罗列。

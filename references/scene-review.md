# 评价分册（候选人 / 晋升 / 同事评价）

## 结构

**结论先行 + 编号短句支撑**：

```
结论：{评级或决策术语} —— {一句话原因}。
整体评价：{正面画像，顿号连}；{主要短板 + 归因}。
1、{维度}：{优秀/良好/一般}。{事实支撑}。
2、...
```

- 看潜力型（校招/实习/培养岗）按能力维度组织，每维度标三档（优秀/良好/一般），"优缺点都突出"时不强行打档，双面写
- 资深补位型直接写达标/不达标判断，不写安置建议
- 结论用标准招聘术语（推荐通过/边界/不推荐），不自造词

## 措辞铁律

1. **推理过程不写进结论**：删"反映 X""暗示 Y""需要被引导才能 Z"——直接给判断
2. **措辞克制**：避免过誉（稀缺/突出→不错/偏上）、绝对化（实战为零→实战不足；除非本人当面承认）、主观色彩词（真 owner/心力不够→中性描述）
3. **原话不加引号转述**（除非是关键短板的当面承认）
4. **不带批次比较**：评价只写绝对值描述，横向对比留在口头
5. **不写入职后操作性安排**：mentor 配置、带教计划、转正节点都是用人方的事
6. **短板归因升维 + 同源收束**：多个具体短板统一到更本质的维度；校招生短板归因到"阶段性场景受限"而非个人能力
7. **证据强度看回答质量不看流程完整度**：问了但答不出具体，本身就是结论性证据——"该强而不强"（自我标榜的优势维度上给抽象答案）权重最高，不要用"面试没走完"为空洞回答开脱

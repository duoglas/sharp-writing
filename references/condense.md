# 构建与缩写分册（概括生成）

本 skill 有两个方向相反的分支：**审读分支**（三查、五关、词表）拿已有句子往回验，只能拦住坏概括；**构建分支**（本分册）解决怎么产生好概括——写新内容和缩写已有内容都走这里。

核心原理一句话：**概括是归纳的产物，不是压缩的技巧**。高度概括 ≠ 抽象词多，范本（大厂汇报稿 / 讨论型 PPT / 周枫文章）的高浓度全部来自"判断 + 数字"，不是来自抽象名词。

## 归纳跳跃：概括的唯一合法产物是新判断句

金字塔原理的可操作内核就这一条：一组事实向上提炼时，产出必须是一个**新的判断句**，不是**分类标签**。

- "三个积极信号" / "业务结果" / "研发效能情况" —— 标签，读者获得零信息
- "结构转型已启动但进展偏慢" / "任务层跑稳，瓶颈转移到跨角色环节" —— 判断，替读者完成了一次思考

判别法：读者只看这句话、不看下面的事实，能获得新信息吗？分类名 = 零信息 = 没有完成归纳跳跃。

最常见的失败模式：拿组织现成的分类当概括——按 KR 编号、按部门、按时间线归组，然后组名冒充结论。那是别人的框架，不是你的思考。

## 判断句的形态：写正题，不写反题

归纳跳跃的思考过程天然是"推翻旧读法、立起新读法"，直接誊写就会得到二元对比骨架——"不是 X 而是 Y""不在 X 在 Y""不再是 X，Y 才是"，以及缩写变体"X 而非 Y""并非 X"（2026-07-07 实证：变体照样是残渣，且更容易漏检）。这是 AI 模板句式（清理层必删项），也是推理残渣：反题是你思考时的稻草人，读者不需要它（同源规则：人员评价的"推理过程不写进结论"）。

- 写法：**正面陈述新判断 + 冒号 + 数字证据**。对比信息由数字并置承载，不由句式承载——"任务 96.8% 对需求 71.5%"自己会说话，不需要"不再是瓶颈"来宣布
- 反面："单点执行不再是瓶颈，跨角色环节是" → 正面："执行瓶颈已上移到跨角色环节：任务按时率 96.8%，需求 / 测试 / 验收全卡在 70% 上下"
- 反面："AI 收入的瓶颈不在演示在验收" → 正面："AI 收入卡在验收转化段：90% 的回款来自已过验收线的两个产品"
- 例外：反题是读者的**活跃误解**、不纠偏会导致错误决策时，可保留显式否定，全篇最多 1 处（与对仗查同额度）

## 写新内容：四步归纳（自下而上金字塔）

1. **铺开**：把全部事实带数字列出来，先不分组。此时禁止写任何判断
2. **归组**：按隐藏的共同原因重新归组，组间不重不漏。警惕直接沿用材料自带的分组（KR 编号 / 部门 / 时间线）——沿用现成分组 = 跳过了自己的思考
3. **组内跳跃**：每组问一次"这堆事实共同说明了什么"，逼自己写出一个判断句（不是组名）。例：96.8% 任务按时率 + 71.5% 需求按时率，两个数没有直接说话，"任务层跑稳、瓶颈转移到跨角色环节"是归纳出来的
4. **顶层收口**：对三四个组判断再问一次"合起来意味着什么"，得到中心判断——它就是 PLAN 三行里的主要矛盾行

## 第一性遮盖检验（对中心判断）

让中心判断从目标推导出来，而不是从材料结构里挑出来。检验方法：

把 KR 编号、部门名、上期汇报的框架**全部遮掉**，只留两样东西——"读者要做什么决策" + 裸事实。你的中心判断还成立吗？

- 成立 → 它是从目标推导出来的，过
- 没了着落 → 它是从旧框架继承的，回到第 2 步重新归组

## 缩写已有内容：详→简三问

压缩 = **换层级重写**，不是删字。删字丢的是 so what 和基线，产物是电报腔；换层级丢的是次要细节，产物是"更短但同样具体"。对详细版的每一段问三问：

1. **判断问**：这堆事实共同说明的一句话判断是什么？（必须是新信息，不能是分类名）
2. **验证问**：保留哪 2 个数字，能让读者自己验证这个判断？
3. **决策问**：这个判断改变了读者的哪个决策？（so what 不能在压缩中丢掉）

概括句的标准形态 = **判断 + 2 个数字 + so what**。写不出判断问的段落说明详细版本身没想清楚，回去补思考再压。

## 与审读分支的关系

归纳是正向发动机，三查是反向刹车：本分册生成的概括句，仍然要过概括句三查（反推 / 对仗 / 基线）和分册自检——生成和校验闭环，缺一边都会漏。

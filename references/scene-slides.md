# PPT 分册

页标题的完整规则（含文档/文章标题）见 [titles.md](./titles.md)。

先判类型：**讨论驱动型**（评审会、方向讨论、workshop）还是**汇报驱动型**（述职、进展汇报）。两型规则不同。

## 通用规则

1. **一页一个判断**：每页只回答一个问题；标题写结论句不写主题词（"成本下降 40%"不是"成本分析"）
2. **标题连读成线**：全部页标题抄出来连读，应该是一条完整叙事线；断了就是缺页或废页
3. **眯眼测试**：眯眼看每页轮廓，视觉重心必须落在关键信息上；3 秒抓不到重点的页重做
4. **数字翻译成体感**：对跨级/外部读者，大数字配一个生活尺度参照（"8GB = 1000 首歌"）；对专业读者配 vs 基线。两者可并用
5. **三段律**：叙事主干不超过三件事，超了合并

## 讨论驱动型

- 页标题用短锚点（可英文：The "Why" / Current Situation / Next Stage），锚点串起议程
- 每页字极少、判断密度高——文字是讨论锚点不是讲稿
- 一张框架图贯穿全场，用增量标注表达"现状 → 下一步"（红笔圈现状、箭头画方向），不换图
- 证据用原件截图直接成页，关键词高亮，不做转述摘要页
- 用问题推进讨论，且问题自带判断标准（"编译快 50% 还是 P99 降 20%？必须有量化指标"）
- 最后一页给产出物模板（空表格），会前定好会议 Output 格式

## 汇报驱动型

- 首页 dashboard（同汇报分册）；后续页按目标线展开
- 每页标题 = 该页结论；页内数据带 vs；好坏对称呈现
- 附录放 backup 数据页，正文页数克制

## 自检表

- [ ] 每页标题是判断句；标题连读成线
- [ ] 每页只有一个判断；无中药铺式 bullet 页
- [ ] 数字有基线或体感参照
- [ ] 讨论型：框架图贯穿 + 有 Output 模板页；汇报型：首页 dashboard 成立
